## 🔧 **Maintenance Guidelines**

### **When to Create Documents**
- 🔄 **Major Refactoring**: Any significant code restructuring
- 🚨 **Critical Bug Fixes**: Issues affecting core functionality
- 🏗️ **Architecture Changes**: Design pattern modifications
- ⚡ **Performance Optimizations**: Measurable improvements
- 📊 **Technical Debt**: Systematic debt identification and management

### **Document Lifecycle**
1. **Active Phase**: Document created during development
2. **Review Phase**: Technical review and validation
3. **Archive Phase**: Move to Archive/ when superseded
4. **Reference Phase**: Historical context for future decisions

### **🔄 Technical Debt Review Cycle** ⭐ NEW
- **Weekly**: Progress updates on active refactoring initiatives
- **Monthly**: Full priority matrix review and planning (use MonthlyReviewTemplate.md)
- **Quarterly**: Strategic review and long-term planning
- **Ad-hoc**: Emergency technical debt assessment for critical issues
